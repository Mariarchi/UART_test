# 1. Подготовка:
### 1.1 Обнаружить устройство в терминале:
`ls -l /dev/ttyUSB\*`

### 1.2 Чтобы каждый раз не вводить sudo, надо добавить своего пользователя в группу dialout:
`sudo usermod -aG dialout $USER`

### 1.3 Текущие настройки порта проверяем так:
`stty -F /dev/ttyUSB0 -a`

# 2. Пересылка и приём данных по UART
### 2.1 Отправка данных (запись в порт):
`echo "Hello UART" > /dev/ttyUSB0`

### 2.2 Чтение данных (вывод на экран):
`cat /dev/ttyUSB0`

# 3. Более детальная настройка STTY:

### 3.1 Пример настройка скорости порта:
`stty -F /dev/ttyUSB0 115200 raw -echo`

### 3.2 Настройка и конфигурация порта:
Примеры контроля чётности:
Передатчик:

`stty -F /dev/ttyUSB0 raw 9600 cs7 parenb parodd -cstopb -crtscts -echo`

Приёмник:

`stty -F /dev/ttyUSB1 raw 9600 cs7 parenb -parodd -cstopb -crtscts -echo inpck parmrk -ignpar`

Когда вы пишете **raw**, Linux автоматически подставляет за вас внушительный список параметров, переводящих порт в сырой бинарный режим. Raw по сути эквивалентен ручному вводу параметров: -ignbrk -brkint -ignpar -parmrk -inpck -istrip -inlcr -igncr -icrnl -ixon -ixoff -icanon -opost -isig -iuclc -ixany -imaxbel -xcase min 1 time 0



|Конфигурация                           |Флаги                                                                                                                         |
|---------------------------------------|------------------------------------------------------------------------------------------------------------------------------|
|Количество битов данных                |**cs8** — 8 битов (используется чаще всего); **cs7** — 7 битов; **cs6** — 6 битов; **cs5** — 5 битов                                          |
|Контроль чётности (Parity)             |**-parenb** — без чётности (None); **parenb -parodd** — чётность (Even); **parenb parodd** — нечётность (Odd)                             |
|Стоповые биты (Stop Bits)              |**-cstopb** — 1 стоповый бит; **cstopb** — 2 стоповых бита                                                                            |
|Управление потоком (Flow Control)      |**-crtscts** — отключить аппаратное управление (Hardware Flow Control); **-ixon -ixoff** — отключить программное управление (XON/XOFF)|
|Обязательные флаги для бинарного режима|**raw** — отключает обработку спецсимволов Linux; **-echo** — отключает эхо (данные не возвращаются)                                  |
|Обработка ошибок чётности              |**-inpck** (по умолчанию) — Игнорировать проверку; **inpck parmrk -ignpar** — Маркировать ошибки. Повреждённый байт не выбрасывать; **inpck -parmrk ignpar** — Все байт с нарушенной чётностью стираются из буфера; **inpck -parmrk -ignpar** — Повреждённый байт заменяется на нулевой     |


# 4. Как правильно сгенерировать 2 КБ чистых данных для теста под каждый режим:
### 4.1 Генерируем 2 кБ для режима cs5:
`od -An -v -tu1 -N 2048 /dev/urandom | awk '{for(i=1;i<=NF;i++) printf "%c", $i%32}' > test_cs5.bin`
### 4.2 Генерируем 2 кБ для режима cs6:
`od -An -v -tu1 -N 2048 /dev/urandom | awk '{for(i=1;i<=NF;i++) printf "%c", $i%64}' > test_cs6.bin`
### 4.3 Генерируем 2 кБ для режима cs7:
`od -An -v -tu1 -N 2048 /dev/urandom | awk '{for(i=1;i<=NF;i++) printf "%c", $i%128}' > test_cs7.bin`
### 4.4 Генерируем 2 кБ для режима cs8:
`dd if=/dev/urandom of=test_cs8.bin bs=1 count=2048 status=none`
